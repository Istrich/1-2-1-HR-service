# План: Windows desktop-обёртка для web-приложения

- **Дата:** 2026-04-08  
- **Статус:** approved  
- **Связанное ТЗ:** Нет (локальное инфраструктурное улучшение)

## Цель

Добавить возможность запуска HR 1-2-1 Web как desktop-приложения на Windows в формате `.exe`, не меняя текущий режим эксплуатации через `python app.py` и Docker.

Результат: отдельная точка входа для desktop-режима и инструкции сборки через PyInstaller.

## Текущее состояние

Сейчас приложение запускается как FastAPI-сервер (`python app.py`) и открывается в браузере. Отдельного desktop launcher нет, сборка под Windows `.exe` не документирована.

## Предлагаемое решение

Добавить отдельный файл `desktop_main.py`, который:
- поднимает текущий FastAPI-приложение локально на `127.0.0.1:8080`,
- ожидает готовность порта,
- открывает UI в нативном окне через `pywebview`.

Это изолированный путь: существующий запуск через `app.py` остаётся прежним. Для сборки использовать PyInstaller в режиме `onedir`, чтобы сохранить доступ к `static/` и `outputs/`.

## Риски

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Не попадут статические ресурсы в сборку | Средняя | Высокое | Явно добавить `static` и `outputs` в команду PyInstaller |
| На Windows отсутствует WebView2 runtime | Низкая | Среднее | Добавить в инструкцию проверку/установку WebView2 |
| Поведение отличается от browser-режима | Низкая | Низкое | Desktop-режим только оборачивает текущий localhost UI без изменения API |

## Миграции и данные

Нет. Форматы данных и структура `outputs/` не меняются.

## Rollback

Удалить `desktop_main.py`, `requirements-desktop.txt` и разделы документации по desktop-сборке. Веб-режим `python app.py` остаётся рабочим независимо от desktop-изменений.

## Шаги внедрения

1. Добавить `desktop_main.py` с запуском Uvicorn в отдельном потоке и открытием окна `pywebview`.
2. Добавить `requirements-desktop.txt` для desktop-зависимостей.
3. Обновить README и `context.md` инструкциями по Windows `.exe`.
4. Проверить, что обычный запуск (`python app.py`) не изменился.
